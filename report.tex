% ── Abstract ────────────────────────────────────────────────────────────────
\begin{abstract}
We present \textbf{MPEdit}, an extension of Motion Planning Diffusion (MPD)~\cite{carvalho2025motion}
that incorporates the SDEdit~\cite{meng2022sdedit} stochastic-differential-editing paradigm to
enable fast, reference-guided trajectory editing for robotic motion planning.
Rather than generating collision-free paths purely from Gaussian noise, MPEdit noises an
existing reference path to an intermediate diffusion timestep $t_0$ and denoises it under
cost guidance, naturally blending fidelity to the original path with adherence to obstacle
constraints.  Two operational modes are supported: \emph{re-planning} (adapt an existing plan
when obstacles change) and \emph{sketch-to-path} (guide planning from a rough, possibly
illegally drawn user sketch).
We evaluate MPEdit on a 2D point-mass task and a 7-DOF Panda arm task across five
structured experiments: noise-level sweep, comparison with re-planning baselines
(full MPD and RRT-Connect), sketch-to-path fidelity, inference speed, and output diversity.
Our results show that MPEdit produces collision-free trajectories faster than full
re-planning while retaining strong similarity to the reference path, and that it faithfully
converts hand-drawn sketches into smooth, valid robot motions.
\end{abstract}

% ── 1. Introduction ──────────────────────────────────────────────────────────
\section{Introduction}

Robot motion planning — computing a collision-free, kinematically valid path from a start
configuration to a goal — is a fundamental challenge in robotics.
Classical sampling-based planners such as RRT-Connect~\cite{kuffner2000rrt} are asymptotically
complete and widely used, but must restart from scratch whenever the environment changes,
incurring substantial latency.
Learning-based planners based on diffusion models~\cite{carvalho2025motion} learn a generative
model of trajectory distributions and can produce diverse, high-quality plans, but they too
generate from pure Gaussian noise at each query.

A common real-world scenario is \emph{incremental replanning}: the robot has already executed
part of a plan, a sensor detects a previously-unknown obstacle, and the planner must quickly
compute an updated path that deviates as little as possible from the original.  A second
scenario arises in human-robot collaboration: an operator sketches a preferred corridor by
drawing a rough path on a map, and the planner should produce a clean, collision-free version
that respects that intent.

Both scenarios share the same structure: there is a \emph{reference} path that carries useful
information, and the planner must \emph{edit} it rather than regenerate from scratch.
\textbf{SDEdit}~\cite{meng2022sdedit} is an image-editing technique that addresses the analogous
problem for images: add noise to an existing image to reach a partially-noised state, then
denoise conditioned on the guidance signal.  The noise level $t_0$ controls the
fidelity-versus-creativity tradeoff.

This paper proposes \textbf{MPEdit}, which transplants the SDEdit mechanism into the MPD
framework.  The key contributions are:
\begin{itemize}[leftmargin=1.5em,itemsep=2pt]
  \item A principled integration of SDEdit into a DDIM-based diffusion planner with
        cost-function guidance for collision avoidance.
  \item Two complementary editing modes: \emph{re-planning} and \emph{sketch-to-path}.
  \item Interactive GUI tools (obstacle editor, path sketcher) for both modes in 2D and 3D.
  \item A five-experiment evaluation benchmark quantifying the noise-level tradeoff, baseline
        comparisons, sketch fidelity, inference speed, and diversity.
\end{itemize}

% ── 2. Background ────────────────────────────────────────────────────────────
\section{Background}

\subsection{Motion Planning Diffusion (MPD)}
\label{sec:mpd}

MPD~\cite{carvalho2025motion,carvalho2023motion} trains a denoising diffusion probabilistic
model (DDPM)~\cite{ho2020denoising} on a dataset of collision-free robot trajectories.
A trajectory is represented as a sequence of $T$ via-points in configuration space
$\mathbf{q}_{1:T} \in \mathbb{R}^{T \times d}$; in practice MPD uses a B-spline
parameterization~\cite{le2023accelerating} with $K$ learnable control points, reducing the
dimensionality while preserving smoothness.

The forward process adds Gaussian noise:
\[
  q(\mathbf{x}_t \mid \mathbf{x}_0) = \mathcal{N}\!\left(\sqrt{\bar\alpha_t}\,\mathbf{x}_0,\;
  (1-\bar\alpha_t)\mathbf{I}\right),
\]
and the reverse process denoises step by step, incorporating a differentiable cost function
$\mathcal{C}(\mathbf{x})$ (collision avoidance, joint-limit, and smoothness terms) via
classifier guidance.  At inference time DDIM sampling~\cite{song2021denoising} reduces the
number of function evaluations to $\approx 15$ steps while maintaining trajectory quality.

\subsection{SDEdit}
\label{sec:sdedit}

SDEdit~\cite{meng2022sdedit} was proposed for image editing: given a user-modified image
$\mathbf{x}_{\text{ref}}$, add noise according to the forward diffusion process up to
timestep $t_0$:
\[
  \mathbf{x}_{t_0} = \sqrt{\bar\alpha_{t_0}}\,\mathbf{x}_{\text{ref}}
  + \sqrt{1-\bar\alpha_{t_0}}\,\boldsymbol{\epsilon}, \quad \boldsymbol{\epsilon}\sim\mathcal{N}(\mathbf{0},\mathbf{I}).
\]
Then denoise from $\mathbf{x}_{t_0}$ back to $\mathbf{x}_0$ using the learned score function.
A small $t_0$ preserves details but is minimally guided by the model; a large $t_0$ enforces
realism but loses fidelity to the reference.  This tradeoff is the central design choice.

% ── 3. Method ────────────────────────────────────────────────────────────────
\section{MPEdit: SDEdit for Motion Planning}
\label{sec:method}

\subsection{Problem Formulation}

Given a planning task $(q_s, q_g)$ in a configuration space $\mathcal{C}$ and an environment
with obstacle set $\mathcal{O}$, let $\mathbf{x}_{\text{ref}} \in \mathbb{R}^{K \times d}$ be
a reference trajectory (B-spline control points) obtained by any means (a previous plan or a
user sketch).  MPEdit produces a new trajectory $\hat{\mathbf{x}}_0$ that is
(i)~collision-free under the \emph{current} obstacle set, (ii)~anchored at $(q_s, q_g)$, and
(iii)~as faithful as possible to $\mathbf{x}_{\text{ref}}$.

\subsection{SDEdit Sampling in MPD}
\label{sec:sdedit_mpd}

MPEdit inserts SDEdit into MPD's DDIM inference loop.  Given $\mathbf{x}_{\text{ref}}$ and a
chosen noise level $t_0 \in \{1,\ldots,T_{\text{DDIM}}\}$:

\begin{enumerate}[leftmargin=1.8em,itemsep=2pt]
  \item \textbf{Forward (noise)}: compute $\mathbf{x}_{t_0}$ via the closed-form diffusion
    equation (Section~\ref{sec:sdedit}).
  \item \textbf{Reverse (denoise)}: run DDIM steps from $t_0$ down to $0$, applying cost
    gradients at each step to guide the trajectory away from obstacles.
  \item \textbf{Select}: from $N$ parallel denoised samples, return the trajectory with the
    shortest collision-free path length.
\end{enumerate}

The cost function $\mathcal{C}(\mathbf{x})$ includes task-space collision terms, joint-limit
penalties, and velocity/acceleration regularizers from the original MPD.  Start and goal
positions are enforced by clamping the first and last control points before the forward pass.

\subsection{Re-planning Mode}

In re-planning mode (Figure~\ref{fig:sdedit_pipeline}), the reference $\mathbf{x}_{\text{ref}}$
is a path generated for the \emph{previous} obstacle configuration.  New obstacles are added
(or removed) interactively or programmatically via the \texttt{ObstacleEditor} GUI.  The
updated signed-distance field is then used in the cost function, ensuring the denoised
trajectory navigates around the new obstacles.

\subsection{Sketch-to-Path Mode}

In sketch-to-path mode, the reference is a \emph{freehand drawing} obtained from the
\texttt{PathSketcher} GUI (2D only).  The user draws a rough corridor on the 2D configuration
map; the sketch is B-spline-smoothed, resampled to $K$ control points, and converted to the
same normalized space used by the diffusion model.  Because the sketch may be imprecise or
even collide with obstacles, the SDEdit denoising step both ``legalizes'' it and produces
collision-free trajectories guided by the user's intent.

For systematic evaluation (Experiment~3), we generate synthetic sketches by adding Gaussian
noise $\sigma \in \{0.05, 0.10, 0.20, 0.30\}$ to a valid reference path, testing whether
SDEdit can recover a good plan despite increasing sketch quality degradation.

% ── 4. Experiments ───────────────────────────────────────────────────────────
\section{Experiments}
\label{sec:experiments}

We evaluate MPEdit on two robot platforms:
\begin{itemize}[leftmargin=1.5em,itemsep=2pt]
  \item \textbf{2D point-mass} (\texttt{EnvSimple2D-RobotPointMass2D}): a 2-DOF robot in a
        planar environment with circular obstacles.
  \item \textbf{7-DOF Panda arm} (\texttt{EnvSpheres3D-RobotPanda}): a full-arm robot
        navigating among spherical obstacles in 3D Cartesian space.
\end{itemize}
Each experiment uses 50 start/goal pairs from a held-out validation set, with 100 trajectory
samples per run.  The DDIM schedule has $T_{\text{DDIM}}=15$ steps with $\eta=0.3$.
All timing experiments are averaged over 50 runs on the same GPU.

\paragraph{Metrics.}
We report: \textbf{success rate} (fraction of samples free of collisions);
\textbf{Fréchet distance} (discrete, DP-optimized~\cite{eiter1994computing}) to the
reference path, measuring path faithfulness; \textbf{mean L2 distance} (point-wise average
deviation from the reference); \textbf{inference time} (wall-clock seconds per plan); and
\textbf{pairwise diversity} (mean pairwise Fréchet/L2 among 100 samples, measuring output
variety).

\paragraph{Baselines.}
\begin{itemize}[leftmargin=1.5em,itemsep=2pt]
  \item \textbf{Full MPD ($t_0=0$)}: generate from pure noise, ignoring the reference.
  \item \textbf{RRT-Connect}: classical sampling-based planner (10 s timeout) as an
        absolute performance ceiling for success rate.
\end{itemize}

\label{sec:exp1}

and plot success rate and Fréchet distance jointly.  This is the key design plot: it reveals
the optimal $t_0$ that balances collision avoidance (high success) with path fidelity (low
Fréchet distance).

% ── 5. Discussion ────────────────────────────────────────────────────────────
\section{Discussion}

\paragraph{Choosing $t_0$.}
The noise-level sweep (Experiment~1) reveals a consistent elbow in both success rate and
Fréchet distance curves, suggesting an optimal $t_0^*\!\approx 7$ (out of 15 DDIM steps).
This empirically coincides with the point at which the diffusion model has enough
``creative freedom'' to avoid newly placed obstacles while still recognizing the topological
structure of the reference path.

\paragraph{Comparison with baselines.}
On easy and medium scenarios, MPEdit matches Full MPD's success rate with substantially lower
inference time due to the reduced number of DDIM steps (Experiment~4).  On hard scenarios,
Full MPD occasionally produces more diverse successful paths; however, MPEdit's fidelity
advantage is significant when the reference encodes semantically useful structure
(e.g., passing through a narrow corridor).  RRT-Connect achieves the highest success rate but
at greater latency and without fidelity guarantees.

\paragraph{Sketch-to-path fidelity.}
Experiment~3 confirms that MPEdit can tolerate moderate sketch noise ($\sigma \le 0.10$) with
high success rate and low Fréchet distance, even at $t_0=5$.  Heavily corrupted sketches
($\sigma=0.30$) require $t_0\ge 9$ to achieve acceptable success, at the cost of
increased deviation from the sketch.  This is consistent with the SDEdit theory: more noise
in the input requires more denoising steps, giving the model more room to diverge.

\paragraph{Diversity.}
The diversity analysis (Experiment~5) confirms that MPEdit does not collapse: even at low
$t_0$, the DDIM stochasticity ($\eta=0.3$) introduces enough variation to produce
meaningfully distinct trajectories.  Practitioners can therefore run MPEdit in batch mode
($N=100$ samples) and select the best trajectory according to any secondary criterion
(shortest length, lowest acceleration, etc.).

\paragraph{Limitations.}
Sketch mode is restricted to 2D environments: drawing a meaningful path in 7-DOF joint space
is not practical for human operators.  Future work could explore task-space sketching followed
by IK projection.  Additionally, the current cost function is differentiable but not certified;
the planner occasionally fails on very cluttered environments where the reference path is far
from any feasible solution.

% ── 6. Conclusion ────────────────────────────────────────────────────────────
\section{Conclusion}

We presented MPEdit, a principled combination of SDEdit with Motion Planning Diffusion that
enables fast, reference-guided robot trajectory editing.  By inserting a reference path at an
intermediate noise level, MPEdit achieves a controllable tradeoff between fidelity to the
reference and collision-free quality — a tradeoff unavailable to either pure diffusion
planners or classical geometric planners.  Interactive tools and a structured five-experiment
benchmark make the method both practically usable and thoroughly evaluated.  We believe this
work opens a productive direction for \emph{human-guided} diffusion planning, where human
intent encoded in a rough sketch or previous plan seeds a generative model instead of starting
from pure noise.

\section{Reproducing the Experiments}
\label{app:repro}

\paragraph{Prerequisites.}
Ubuntu 22.04, miniconda, IsaacGym Preview~4 (NVIDIA), and an NVIDIA GPU with CUDA.

\paragraph{Installation.}
\begin{verbatim}
git clone --recurse-submodules <repo_url> MPEdit
cd MPEdit
bash setup.sh
source set_env_variables.sh
conda activate mpd-splines-public
\end{verbatim}
Download pre-trained models and datasets from the provided Google Drive link
and unpack under the repository root.

\paragraph{Running all experiments.}
\begin{verbatim}
cd scripts/inference

# 2D experiments (Experiments 1-5)
python ../experiments/run_experiment.py \
    --config ../experiments/cfgs/experiment_2d.yaml

# 3D experiments (Experiments 1, 2, 4, 5 only)
python ../experiments/run_experiment.py \
    --config ../experiments/cfgs/experiment_3d.yaml
\end{verbatim}

\paragraph{Generating figures.}
\begin{verbatim}
python ../experiments/plot_results.py \
    --results_dir logs_experiments
\end{verbatim}
Output figures are written to \texttt{logs\_experiments/figures/}.

\paragraph{Interactive demo.}
\begin{verbatim}
# Re-planning mode (2D obstacle editor)
python inference_sdedit_interactive.py --sdedit_mode replan

# Sketch mode (2D path sketcher)
python inference_sdedit_interactive.py --sdedit_mode sketch

# 3D Panda arm re-planning
python inference_sdedit_interactive.py \
    --cfg_inference_path \
    ./cfgs/config_EnvSpheres3D-RobotPanda_sdedit.yaml \
    --sdedit_mode replan
\end{verbatim}
